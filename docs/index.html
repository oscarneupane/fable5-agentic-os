The pipeline

Five specialists, one orchestrator

Most AI tools are one model in one shot. Fable5 is a crew — each subagent has a single job, and an independent validator can send the work back for a revision.

🧭

Planner

Breaks the request into small, checkable steps.

🔎

Researcher

Gathers context; marks verified vs assumption.

📐

Designer

Picks the simplest architecture that works.

⌨️

Coder

Writes actual runnable files to disk.

✅

Validator

Self-checks everything; can request a revision.

Researcher and designer run in parallel · the coder waits for both.

Why it's different

Reliable by design

🧱

Structured, not rambling

Every agent returns a typed result with an explicit confidence and its assumptions — uncertainty is shown, not hidden.

🛡️

Built-in critic

Deterministic self-checks run on top of the model and can block approval. A model can't approve away a real problem.

⌨️

Writes real code

The coder writes runnable files to a sandboxed folder — then you run them. Not just descriptions.

⚡

Parallel subagents

Researcher and designer run concurrently on LangGraph for a real speedup.

💸

Per-agent models

Free/cheap model for light work, a strong model for the hard steps. Mix providers per agent.

🔌

Offline fallback

No API key? It still runs deterministically and tells you it didn't verify facts — instead of inventing them.

Get started

Running in under a minute

Clone it, drop in a free GLM key, and go. No key yet? It still runs offline.

# 1. clone & install
git clone https://github.com/oscarneupane/fable5-agentic-os
cd fable5-agentic-os
pip install -r requirements.txt

# 2. add a free key (get one at z.ai or open.bigmodel.cn)
cp .env.example .env        # then set GLM_API_KEY=...

# 3. run — watch the agents work
python cli.py
